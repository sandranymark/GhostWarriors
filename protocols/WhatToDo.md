```

1. Sprint Planning (måndag 11/11)
Eftersom ni startar sprinten idag, håll ett Sprint Planning-möte på förmiddagen. Under detta möte ska ni:

Definiera sprintmålen.
Lista user stories och uppgifter som ni prioriterar att arbeta med under sprinten.
Identifiera eventuella hinder och resurser som kan behövas.
Dokumentera allt i en SprintPlanning.md-fil.
Fil: SprintPlanning.md

```

```
2. Daily Scrum (dagligen måndag - fredag)
För dagliga scrums, håll ett kort möte varje morgon för att:

Uppdatera vad varje teammedlem gjorde igår, vad de ska göra idag, och om det finns några hinder.
Protokollför varje möte kortfattat i en DailyScrum.md-fil, med datum för varje dag.
Fil: DailyScrum.md

```

```
3. Sprint Review (fredag 15/11)
På fredag, när sprinten är avslutad, håll ett Sprint Review-möte med produktägaren och eventuella andra intressenter för att:

Demonstrera funktionerna och resultatet av sprinten.
Få feedback från produktägaren och andra intressenter.
Notera vad som blev klart, vad som inte blev klart och eventuella beslut kring feedbacken.
Dokumentera mötet i en SprintReview.md-fil.
Fil: SprintReview.md
```

```
4. Sprint Retrospective (fredag 15/11, direkt efter Sprint Review)
Efter Sprint Review, ha en Sprint Retrospective med teamet där ni:

Reflekterar över vad som gick bra och vad som kan förbättras.
Diskuterar konkreta åtgärder som ni kan ta med till nästa sprint.
Notera alla punkter och beslutsåtgärder i en SprintRetrospective.md-fil.
```
